Mantener un registro de cambios con herramientas como Git es fundamental en cualquier proyecto de software porque permite saber exactamente qué se cambió, quién lo cambió y cuándo. Esto es especialmente útil cuando algo deja de funcionar, ya que se puede revisar el historial y volver a una versión anterior sin perder todo el trabajo. Además, en equipos donde varias personas trabajan al mismo tiempo, Git permite que cada uno trabaje en su parte sin pisar el trabajo del otro. También sirve como respaldo, porque el código no solo vive en una computadora sino en un repositorio remoto al que se puede acceder desde cualquier lugar. Con cada commit queda documentado el progreso del proyecto, lo que facilita entender cómo evolucionó el código con el tiempo. En resumen, Git no es solo una herramienta técnica, es una forma de trabajar con orden, seguridad y colaboración, tres cosas que cualquier proyecto de software necesita para funcionar bien.
